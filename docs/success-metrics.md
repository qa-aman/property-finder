# Success Metrics

**Date:** 24-05-2026
**Status:** Draft for the POC

The POC proves one thing: it produces a confident, narrowed-down shortlist for a
city the user does not know.

## POC success signals

- **Shortlist quality:** For a test brief (office + mode + budget + preferences), the tool returns 5 areas with named societies, each with a commute estimate and a sentiment summary.
- **Decision confidence:** A test user can say "I would actually visit these" for the majority of the shortlist.
- **Filter value:** The shortlist lets the user reject options before visiting (the level-1 filter), reducing the number of in-person visits needed.

## Explicitly out of scope for the POC

- Revenue metrics and monetization.
- Guaranteed data quality for every city. The POC is city-agnostic (city is a per-search input), but coverage varies by city. Validate against at least two cities (e.g. Mumbai and Bangalore).
- Replacing the in-person visit. The visit (day and evening) stays mandatory.
